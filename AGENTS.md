# 数据治理平台原型约定

本项目是数据治理/数据资产管理类静态原型，采用 `index.html` + `css/` + `js/` 的模块化页面结构。所有新增和优化必须贴合现有数据治理平台风格，不另起视觉体系。

项目结构：
- `index.html`：单页原型入口。
- `css/variables.css`、`css/components.css`、`css/layout.css`：全局变量、组件和布局。
- `css/pages/`：页面级样式。
- `js/config.js`、`js/sidebar.js`、`js/router.js`、`js/app.js`：配置、导航、路由和应用入口。
- `js/pages/`：页面级逻辑。
- `js/components/`：复用组件。

原型修改规则：
- 先定位同类页面再修改，例如数据源目录树、元数据搜索、质量报告、稽查任务、数据地图等已有实现。
- 用户说“参考某页面格式”时，优先复用该页面的信息结构、表格列、卡片和交互模式，不理解为合并两个页面功能。
- 数据质量、稽查报告、元数据、数据地图等示例数据必须符合数据资产管理语境；统计数值要自洽，避免通过率、问题记录数、总记录数互相矛盾。
- 导出、下载等原型交互优先做可演示的前端即时生成或提示，不提前生成大量无关文件。
- 不启动服务、不打开浏览器测试，除非用户明确要求。

提交和检查：
- 用户要求提交/部署时使用全局 `git-deploy-update` 技能。
- 默认运行 `git diff --check`。
- 修改 JS 后做轻量语法检查；系统 `node` 不可用时使用 Codex bundled Node。
- 中文显示异常时先用 UTF-8 重读确认。
